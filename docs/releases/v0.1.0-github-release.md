## Body

```markdown
## Summary

`v0.1.0` is the first production-readiness release for Production RAG
Assistant. It includes a local production-style FastAPI RAG backend with
Postgres/pgvector, workspace isolation, document management, chat sessions,
streaming responses, OpenAI-compatible providers, deterministic evals,
observability, async export jobs, deployment documentation, production
configuration preflight, secret manager mapping, and API key role enforcement.

## Highlights

- FastAPI routes for chat, streaming chat, documents, workspaces, sessions,
  export jobs, health, metrics, and the static web UI.
- Postgres and pgvector schema managed through Alembic migrations through
  `0010_create_export_jobs.py`.
- Markdown ingestion, chunking, content hashing, fake embeddings, OpenAI
  embeddings, and chunk reindexing.
- Hybrid retrieval with vector retrieval, sparse retrieval, metadata filters,
  RRF fusion, optional query rewrite, optional session history, and optional
  OpenAI listwise reranking.
- Fake generator and OpenAI Responses API generator with streaming support.
- Citation validation, refusal guards, provider error mapping, latency metrics,
  token metrics, and provider cost estimates.
- Workspace lifecycle management, archived-workspace write guards, bulk
  workspace operations, and workspace operation audit logs.
- API key workspace scoping via `API_KEY_WORKSPACE_ACCESS`.
- API key role enforcement via `API_KEY_ROLES` with `admin`, `operator`, and
  `viewer` roles.
- Async chat log export jobs with worker loop, downloads, retry, stale running
  job recovery, and export file retention cleanup.
- Production-style Dockerfile and Compose stack for API, migrate, Postgres, and
  export worker services.

## Important Configuration

Review these before deployment:

- `API_KEYS`
- `API_KEY_ROLES`
- `API_KEY_WORKSPACE_ACCESS`
- `DATABASE_URL`
- `SYNC_DATABASE_URL`
- `OPENAI_API_KEY`
- provider selection variables
- CORS settings
- rate limit settings
- export worker settings

Use `docs/SECRET_MANAGER_MAPPING.md` for secret classification and
`docs/RELEASE_CHECKLIST.md` for release gates.

## Known Limitations

- API key roles are a production-friendly foundation, not a full user identity
  system.
- In-process rate limiting is intended for local or single-instance deployment.
  Multi-replica deployments should use Redis, an API gateway, or a reverse
  proxy layer.
- The web UI is intentionally minimal and served by FastAPI.
- Real provider quality and cost depend on configured OpenAI-compatible models
  and `PROVIDER_PRICE_TABLE`.

## Rollback

Use `docs/DEPLOYMENT_RUNBOOK.md` for operational rollback commands and
`docs/RELEASE_CHECKLIST.md` for rollback readiness checks.
